HTML

Réaliser le squelette HTML de son application

Retour

  • Créer le dossier de travail

    Je télécharge un boilerplate que je renomme du nom du projet, ou je crée le dossier et les sous-dossiers manuellement

  • Préparer la structure

    Dans index.html je balise les grandes partie de ma page à l'aide de commentaires et je charge les fichiers extérieurs nécessaires

  • Créer les pages

    A la racine du dossier, je duplique la page index.html autant de fois qu'il y a de pages sur le site et je renomme ces pages

  • Coder le template

    Dans index.html et sur la base du wireframe, je code uniquement les éléments du template

  • Vérifier le template

    Je contrôle visuellement dans le navigateur pour toutes les tailles d'écrans que tous les éléments du template sont présents dans le bon ordre et que le template est lisible

  • Valider le template

    Je passe le code de ma page index.html au validateur du W3C.

  • Corriger les erreurs

    Si des erreurs sont présentes, je corrige mon code et je le repasse au validateur

  • Copier le template

    Quand mon template ne contient plus d'erreurs, je le copie sur chaque page du site et je n'y touche plus

  • Coder le contenu principal

    Sur la base du wireframe, je code le contenu principal de chaque page.

  • Vérifier le site

    Je contrôle visuellement que le site est lisible, compréhensible et navigable sur toutes les tailles d'écrans et que je chaque page contient les bons éléments.

  • Valider chaque page

    Je passe le code de chaque page au validateur du W3C

  • Corriger les erreurs

    Si des erreurs sont présentes, je corrige mon code et je le repasse au validateur
